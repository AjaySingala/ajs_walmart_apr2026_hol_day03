Temperature: between 0 and 1.
The more it is towards 0, the more deterministic will be the response of the LLM.
The more it is towards 1, the more creative and varying will be the response of the LLM.

For the same question, you expect the same answer (more or less), the temperature value must be closer to 0.
For variety (creativity, randomness, etc.), the temperature value must be closer to 1.

RAG vs Fine-tuning (re-training a model):
---
RAG: documents


Data/datasets: Excel, csv, tables (DBs), APIs....docs also.
GPT, Claude, Gemini --- LLMs/models --- trained on generic data collected from the internet.
	- they can answer or perform any task
	- generate text, summary, image, email, analyze, ....
	
Specific LLM for our purpose only.
Legal, Medical, Financial....
- They need domain/business specific LLMs
- Need a lot of data to begin with
- Create their own LLMs
	- a lot of resources -> servers, people (with the right expertise), time, money
- Building from scratch is difficult
- Take an existing model, and re-train/fine-tune it on their data.
- Configure the model on their servers (on-prem, cloud) - within their network
	- the data does not go outside the enterprise
- Data cleansing
	- Preparing the data
	- Determine which data points/columns/features to use
	- manage missing/invalid
- Split the data into train, test and validate
- the actual implementation of the NN.
- will never work in the first try
	- retry again and again with different parameter settings etc. till you come with a "comfortble" acceptance that the LLM is working as expected on YOUR DATA!!!!
	
Fine-tuning: define behavior of the model
	- What role to play
	- what output in what format to return back
	- any constraints
